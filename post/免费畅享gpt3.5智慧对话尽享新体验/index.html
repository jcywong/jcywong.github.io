免费畅享GPT3.5，智慧对话，尽享新体验

Apr 5, 2024

背景

4月1号起，ChatGPT允许不登录使用，因此就有了无限可能。

通过2个开源项目（FreeGPT35和ChatGPT-Next-Web），记录搭建过程，分享心得。

前提：

  • 搬瓦工VPS一台(IP必须是可食用ChatGPT)
  • 域名一个(非必须)
  • linux系统：Docker环境、1Panel面板

API

搭建

以下是两种方式搭建，任选其一即可：

  1. 使用docker命令安装：docker run -p 3040:3040 missuo/freegpt35

  2. 使用1Panel面板：按下图设置即可

SSL和反向代理

  1. 在域名商中设置二级域名解析到vps的IP地址上

  2. 设置SSL，按图设置并修改为自己的域名即可

  3. 设置反向代理，按图设置并修改为自己的域名即可

  4. 网站配置启动https, 按图设置即可

Next-Web

搭建

以下是两种方式搭建，任选其一即可：

  1. 直接使用docker命令，修改相应内容

    docker run -d -p 3000:3000 \
   -e OPENAI_API_KEY=gptyyds \
   -e CODE=页面访问密码 \
   -e BASE_URL=http://api的地址端口或域名 \
   yidadaa/chatgpt-next-web

  2. 使用1Panel面板：按下图设置，修改相应内容

SSL和反向代理

  1. 在域名商中设置二级域名解析到vps的IP地址上

  2. 设置SSL，按图设置并修改为自己的域名即可

  3. 设置反向代理，按图设置并修改为自己的域名即可

  4. 网站配置启动https, 按图设置即可

使用

直接使用网页端

  1. 打开网页，登录 密码为next-web中设置的code

  2. 设置接口地址为搭建的API 地址 ，APIkey为任意字符，按下图填写即可

App使用

我测试的是chatX 按图配置即可，也可使用OpenCat, Lobe-Chat, Bob等

总结

通过服务的部署，无需使用官方的token就可以使用3.5，这大大方便了我们，也给我们的钱包减压了，最后希望官方不要朝令夕改，哈哈哈
